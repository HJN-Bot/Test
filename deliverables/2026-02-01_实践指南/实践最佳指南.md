# OpenClaw 实践最佳指南（部署安装 + 工作流 + 技能推荐）

> 目标：用最小成本跑通「输入 → 结构化存储 → 看板可视化 → 定期提醒/复盘 → 产物交付（GitHub）」的闭环。

---

## 0. 你会得到什么（最终形态）

- **Airtable**：你的“大后方”——Ideas / Tasks / Daily / Eurekas / Smart Speaker 等全量数据。
- **Dashboard（网页）**：你的“战情室”——只显示近 14 天 Top 8 Active Threads + 两周 ToDo + 项目时间线。
- **Discord**：你的“输入/提醒通道”——日常把想法丢给我，18:00（每两天）收到精简摘要。
- **GitHub（Bot Repo）**：你的“交付仓库”——我产出的流程图、PPT、文章、学习总结都按日期归档。

---

## 1) 部署安装（最短路径）

### 1.1 服务器（EC2）
- 系统建议：Ubuntu
- 必需：node、openclaw gateway、systemd user services

### 1.2 Browser 能力（可选但强烈建议）
- 用于：自动打开网页、截图、抓内容（配合信息收集/整理）

### 1.3 Airtable 接入（安全第一）
- 用 **PAT**（Personal Access Token），不要在聊天里发。
- token 存在服务器 env 文件里（chmod 600）：
  - `~/.config/systemd/user/airtable-nexus.env`

### 1.4 Dashboard 网页访问（Basic Auth）
- 目标：你手机/电脑直接访问 `http://<EC2_IP>:8080/`
- env：`~/.config/systemd/user/dashboard-web.env`
  - `DASHBOARD_USER=...`
  - `DASHBOARD_PASS=...`

## 2) 日常工作流（建议固定节奏）

### 2.1 日常输入（Discord）
- `/idea ...`：新想法/新学到的东西
- `/task ...`：明确要做的事（可执行、可跟踪）
- `/eureka ...`：感悟/方法论（可复用）

> 规则：**写入 Airtable 前必须二次确认**（我先发拟写入内容，你回“确认”才写）。

### 2.2 看板刷新（Dashboard）
- 每次写入 Airtable 后：自动刷新 `dashboard/index.html`
- 看板只保留：近 14 天、Top 8、低噪音摘要（标题 + 相关 + 状态 + 一句话）

### 2.3 下班提醒（每两天 18:00）
- 我会同步 Airtable → 刷新 Dashboard → 在 Discord 发简短摘要
- 提醒你：是否需要“把 1 条 Idea 转成 Task”或“清理 14 天未动条目”

### 2.4 周日复盘（20:00）
- Weekly Synthesis：本周主题/进度/阻塞/能量审计
- Munger Observer：机会成本、二阶效应、偏差与安全边际

## 3) 产物交付（GitHub Bot Repo）

- 仓库：`HJN-Bot/Test`

  - `deliverables/YYYY-MM-DD_<交付内容>/`
  - 示例：`2026-02-01_流程图/`、`2026-02-01_PPT/`

## 4) 技能推荐（按 ROI 分层）

### 4.1 必备（闭环核心）
- **Airtable + Dashboard 同步脚本**：工作流主干
- **video-transcript-downloader / youtube-watcher**：视频→文字→总结
- **markdown-converter**：万物→Markdown
- **excalidraw-flowchart**：流程图/架构图交付
- **github（gh）**：产物推送到 bot repo

### 4.2 强推荐（效率/质量跃迁）
- **humanizer**：小红书/公众号/演讲稿去 AI 味
- **frontend-design**：dashboard/页面审美与动效
- **deepwork-tracker**：计划 vs 实际的证据链
- **github-pr / conventional-commits**：工程交付规范化

### 4.3 选配（有明确场景再装）
- **nano-banana-pro**：图片生成/编辑（封面、配图、海报）
- **bird**：X/Twitter（发帖需你确认）

---

## 5) 常见坑（避坑清单）

- **不要把 token / 密码发到聊天里**（PAT、Basic Auth、邮箱密码都算）。
- Dashboard 端口尽量用 80/443（公司网络可能会挡 8080），或至少确保安全组放通。
- 任务不要无限堆：看板只留 Top 8，其余放 Airtable，靠周复盘“消化”。
